# מבנה סטטוס ציבורי

קובץ נתונים: `data/status.json`

מטרת הקובץ: לשמור סטטוס כללי לפי השתלמות ומפגש, רק לפי מידע שנמסר במפורש.

## שדות

- `trainingId` — מזהה השתלמות מתוך `data/trainings.json`.
- `meetingNumber` — מספר מפגש, אם הסטטוס שייך למפגש מסוים.
- `date` — תאריך, אם נמסר.
- `kind` — סוג המעקב.
- `status` — סטטוס ברור.
- `source` — מקור המידע.
- `note` — הערה קצרה.

## ערכים מותרים לסטטוס הוראה

- `מלמד`
- `לא מלמד`
- `לא ידוע`

## כלל אמת

אין למלא סטטוס לפי ניחוש. אם אין מקור ברור, כותבים `לא ידוע` או משאירים את הפריט כחסר.
