# כללי מערכת hishtalmut

## 1. מטרת הריפו

הריפו מנהל מידע אמיתי על השתלמויות, מפגשים, תאריכים, שעות, נושאים, קבצים, הערות ונתונים חסרים.

## 2. כלל ברזל

אין דמו.
אין נתונים מומצאים.
אין להמציא מספר השתלמות.
אין להמציא מנחים, קבצים או פרטים שלא סופקו.
אם מידע חסר, מציינים במפורש: חסר / נדרש להשלים.

## 3. מבנה מידע

כל השתלמות נשמרת גם כקובץ JSON לשליפה חכמה וגם כקובץ Markdown לקריאה אנושית.

כל השתלמות חייבת לכלול:
- מזהה פנימי
- מספר השתלמות רשמי אם קיים
- סטטוס מספר השתלמות
- שם השתלמות
- שנה
- מספר מפגשים
- שעות אקדמיות
- תאריכי התחלה וסיום
- רשימת מפגשים
- נושאים
- קבצים ונספחים אם קיימים
- נתונים חסרים

## 4. אתר

האתר חייב להיות:
- עברית מלאה
- RTL מלא
- מותאם לנייד ולמחשב
- כפתורים גדולים
- ניווט נוח
- צבעי פרימיום
- חיפוש חכם
- ייצוא CSV עובד
- העתקת סיכום עובדת
- ללא כפתורים מזויפים

## 5. מקור אמת

הקבצים תחת data הם מקור האמת הטכני.
הקבצים תחת trainings הם מקור אמת אנושי לפי השתלמות.
קובצי STATE מתעדים מצב ועדכונים.
